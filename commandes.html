Commandes Git Courantes

git init
Initialise un nouveau dépôt Git. Cela crée un nouveau sous-dossier .git dans le répertoire courant.
git clone [url]
Clone un dépôt existant à partir de l'URL fournie. Cela copie tous les fichiers du dépôt distant dans un nouveau répertoire local.
git add [fichier]
Ajoute un fichier à l'index, le préparant pour le prochain commit. Utilisez '.' pour ajouter tous les fichiers modifiés.
git commit -m "[message]"
Enregistre les modifications apportées aux fichiers dans le dépôt, avec un message descriptif expliquant les modifications.
git status
Affiche l'état du répertoire de travail et de l'index. Montre les modifications, les fichiers non suivis, et les fichiers prêts à être commités.
git push [alias] [branche]
Pousse les modifications des branches locales vers un dépôt distant. 'alias' est généralement 'origin', et 'branche' est la branche à pousser.
git pull
Tire et fusionne les modifications d'une branche distante dans la branche locale courante.
git fetch
Récupère toutes les branches du dépôt distant mais ne fusionne aucune modification dans les branches locales.
git merge [branche]
Fusionne les modifications d'une branche spécifiée dans la branche active. Utilisé après un 'git fetch' pour combiner les branches distantes.
git stash
Met en réserve les modifications locales non commitées, permettant un changement temporaire de branche sans perdre les modifications.
git log
Affiche l'historique des commits pour la branche courante, listant les commits, les auteurs, les dates et les messages de commit.
